# Règles d’archivage des sprints

- À la fin de chaque sprint, on copie le template dans `sprint-Ix-summary.md` et on le remplit.
- La vélocité ne compte que les tâches en statut **Done**.
- Toute tâche **WIP** est reportée : on met à jour `Sprint` (sprint actuel) et on garde `Sprint initial` inchangé.
- On garde une preuve courte par tâche (commit, capture Sheets, export UML/OCL).
